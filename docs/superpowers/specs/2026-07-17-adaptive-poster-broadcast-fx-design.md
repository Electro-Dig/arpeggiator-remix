# 自适应海报与直播手势音效设计

## 目标

录音开始时保存用户实际看到的完整网页第一帧，保持浏览器当时的真实宽高比；海报只做等比缩放，不裁切、不拉伸。保留现有 Filter、Delay、Glitch 连续控制，再增加两个不会停止节拍的直播安全效果。

## 画面

- 截图 Canvas 使用网页快照本身的宽高，不再固定为 1200×675。
- 原始摄像头仍按网页现场的 `object-fit: cover` 方式铺底，从而与手部轨迹及 HUD 保持对齐。
- 网页快照和 WebGL 图层在同一宽高比画布上叠加，不做二次裁切。
- 海报提供一个最大画面区域，根据源图比例计算实际绘制矩形；源图完整等比放入，周围由海报纸张自然留空。
- 海报参数区根据实际画面高度向下流动，保证 4:3、16:10、16:9 等比例均不重叠。

## 直播手势音效

### Reverb Build

双手同时抬高时，混响量随双手平均高度连续增大；单手演奏不触发。效果松手立即平滑回落，丢失追踪或打开弹窗时归零。

### Drop Impact

双手完成上举蓄力后，在限定时间内同步快速下压，触发一次量化感强但不打断 Transport 的低频 Impact。触发后有冷却时间，不会连续误触；追踪丢失会取消蓄力。

## 反馈与安全

- HUD 的 MIX FX 增加 `RVB` 百分比。
- 控制面板增加两行手势说明。
- Build 与 Impact 使用已有中央状态提示，不新增遮挡人脸的大面积控件。
- Impact 是附加声音，不静音、不降速、不停止节拍；所有连续效果均有失踪复位。

## 验收

- 1440×900 网页快照生成 1440×900 封面，绘制调用不裁掉任何网页像素。
- 海报对 4:3、16:10、16:9 源图均使用完整源矩形。
- 双手抬高能连续得到 0–100% Build；上举后快速下压仅触发一次 Impact。
- 单手、慢速下移、追踪丢失不会误触 Impact。
- 全量测试通过。
