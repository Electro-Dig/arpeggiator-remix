# Exhibition V2 极简视觉精修规格

**状态：** 用户已确认视觉方向，等待书面规格复核

**归档基线：** `exhibition-v2-ui-v1` → `139619e`

**工作分支：** `feature/exhibition-v2`

## 1. 目标

保留当前摄像头画面、顶部 HUD、右上操作按钮、左下社交链接和中央交互区域的整体结构，只通过配色、字级、间距、对齐和少量结构线提升完成度。视觉方向是克制的极简编辑排版，不引入抽象的“精密乐器”装饰体系。

## 2. 非目标

- 不新增扫描线、故障效果、旋钮、插孔、螺丝或设备零件。
- 不增加复杂动画、外部字体、图片素材或运行时视觉库。
- 不改变音乐、手势识别、摄像头、录音或邀请门禁逻辑。
- 不改变主要控件的位置，不遮挡人物和双手交互区域。

## 3. 配色与对比度

- 摄像头仍保持灰度主体。
- 主文字使用偏暖灰白，保证浅色背景下的可读性。
- 青色只用于实时数值、当前状态和主要操作，不让所有文字同时发光。
- 暖橙色只用于录音、步骤编号和现场重要提示。
- 面板背景使用低透明度深炭黑，并通过局部渐变保证文字在明亮摄像头画面上可读。
- 全屏网格降低透明度与密度，不再压住人物面部。

## 4. 字体与编辑排版

- 不加载网络字体。拉丁标签优先使用 Windows 本地 `Bahnschrift` / `DIN Alternate` / `Arial Narrow`，中文使用系统无衬线字体。
- 标签使用窄体大写、小字号和较宽字距；动态数值提高一个字级并减少字距。
- 中文正文使用正常字距和舒适行高，避免把中文强行做成终端字符。
- 文本层级固定为：微型标签 → 主要数值或标题 → 简短说明 → 次要脚注。

## 5. 常驻 HUD

- 左上录音状态保持独立；顶部中央四组 SCENE、SYNTH、RHYTHM、BPM 保持现有位置。
- 每组信息只使用一条短细线和文字对齐，不添加卡片底板。
- 右上按钮保留原位置，减薄边框、缩小内边距，并使用更清晰的中文主标签和英文辅助标签。
- 左下社交链接取消厚重方框，改成低调文字链接与细下划线反馈。
- 简化模式继续隐藏非必要网格和装饰，不影响功能控件。

## 6. 指引卡片

- 保留三张卡片和现有打开、上一张、下一张、跳过行为。
- 去掉圆形装饰和密集卡片网格，改为不对称两栏。
- 左栏显示超大步骤编号 `01`、`02`、`03` 和短标签；右栏显示标题、正文与动作记号。
- 动作记号采用 `L / MELODY`、`R / RHYTHM`、`↑↑ / CONFIRM`、`↓↓ / CANCEL`，不使用 emoji。
- “观众请避免在镜头内举手”作为暖橙色现场脚注，不与主要操作说明争抢层级。
- 底部导航用细线分隔；下一张为主要操作，上一张和跳过保持弱化。

## 7. 调试反馈清理

- 默认隐藏截图中横跨面部的绿色 Delay 横条和紫色 NoteLen 数值标签。
- Delay、NoteLen 等详细数值移入 `CONTROL DECK`。
- 参数变化时允许短暂显示一条细反馈线或一段小型状态文字，使用 `opacity` 和 `transform`，结束后恢复空的 `aria-live` 区域。
- 本轮只改变显示方式，不改变参数计算与音频行为；二维节奏批次仍会按原计划正式移除右手 NoteLen 映射。

## 8. 技术边界

- 主要修改 `styles.css`、`index.html` 和少量 HUD/状态显示代码。
- 不引入构建步骤和新依赖。
- 动画仅使用 `opacity`、`transform`；遵守 `prefers-reduced-motion`。
- 新增样式不能增加每帧 DOM 重排；手部追踪循环中不创建新 DOM 节点。

## 9. 验收

- 自动测试继续覆盖编辑器卸载、社交链接、HUD、简化模式、Control Deck 和 reduced-motion。
- `npm test` 全部通过，`node --check main.js` 通过，`git diff --check` 无输出。
- 在 2048×1055 和常见 16:9 画面中，人物面部和双手区域不被常驻 UI 遮挡。
- 主界面、指引卡片和 Control Deck 由用户截图复核；当前 Codex 浏览器自动控制接口不可用时，使用本地站点加人工截图完成 smoke。
- 初版可随时通过标签 `exhibition-v2-ui-v1` 回看，不覆盖该归档。
