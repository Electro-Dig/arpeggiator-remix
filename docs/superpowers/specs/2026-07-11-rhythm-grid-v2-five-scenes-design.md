# Rhythm Grid V2 与五场景设计

## 目标

在不增加浏览器端模型和现场性能负担的前提下，让右手节奏空间更跟手、49 个节奏具有更清晰的横纵轴听感差异，并把音乐入口精简为五个独立场景。

## 已确认交互

- 右手连续位置使用一个小型青色圆点逐帧显示，不经过 7×7 量化缓动。
- 量化方格继续表示当前已选择并等待/正在生效的节奏单元。
- 横轴语义为 `STRAIGHT → BROKEN`：从规整四拍逐步增加切分、错位和破碎结构。
- 纵轴语义为 `MINIMAL → ENERGY`：从稀疏骨架逐步增加层次、密度与填充。
- 右下角网格保留并略微增大，整体上移以避开 16 步音序器。

## 节奏网格生成

继续离线生成 49 个 JSON 节奏，不在运行时加载机器学习模型。四角仍为 Minimal House、Techno Drive、Electro Break、Glitch Rush，但生成器改为受约束的感知网格：

- 每一行设定逐级上升的目标密度区间，禁止高能区长期停留在相同总击打数。
- 横向通过 kick/snare/clap 的结构位移和切分程度产生差异，而不是只改镲片。
- 纵向除增加 closed/open hat 外，还引入 ghost kick、clap reinforcement 和末拍 fill。
- 所有相邻格满足最小汉明距离；若不足，由确定性变异器补足。
- 保留每步最多三个同时发声轨道、第一拍 kick、基本 backbeat 等现场稳定性约束。

## 五个音乐场景

最终场景固定为：

1. Minimal Groove
2. Groove Pulse（默认）
3. Neon Drive
4. Midnight Pulse
5. Arcade Horizon

Minimal Groove 与 Groove Pulse 直接使用原 Classic 中已确认的旋律、速度和对应 DX7 音色，成为一级场景。删除 Classic 场景、Classic Pattern 下拉框及其余五个旧预设；左手握拳切换当前场景内部音色的逻辑保持不变。

## HUD 与布局

- 顶部五项信息改为权重网格，Scene/Synth/Rhythm 获得更多宽度，BPM/Root 使用窄列。
- 宽屏不再使用省略号；名称在可用空间内完整显示，必要时允许受控换行。
- 节奏网格同时绘制连续圆点与量化方格；只使用 transform 和 CSS，不增加每帧布局计算。
- 小屏继续隐藏顶部状态，并保持原有简化模式行为。

## 验收标准

- 默认打开为 Groove Pulse，场景条恰好五项且无 Classic 下拉框。
- 连续圆点在每次右手更新时移动，量化方格只在节奏格变化时移动。
- 49 个节奏唯一；纵向密度总体严格上升；任意水平/垂直相邻格的完整模式汉明距离达到设定下限。
- 1920×1080 下节奏网格不覆盖 16 步音序器，HUD 三个长名称完整可见。
- 全量自动测试、构建/资源生成和浏览器 smoke 均通过。
