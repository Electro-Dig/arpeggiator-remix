# WAIC 现场卡片、真实计数与 Synthpop 优化设计

日期：2026-07-12  
分支：`feature/exhibition-v2`  
范围：仅测试分支，不部署或合并至生产主站。

## 目标

在不改变现有手势演奏主流程的前提下，完成展览现场可读性、录音确认、二维码分享、活动计数和两段新增音乐场景的优化。所有变化应优先保证现场稳定性，并避免给实时手部追踪与音频线程增加持续性能负担。

## 1. 指引卡片

### 文本可读性

- 放大指引卡正文、辅助说明、底部提示、页码和导航文字。
- 桌面展览视口下，正文目标字号为 20–22 px，辅助文字不低于 16 px；窄屏按比例收缩但不低于 14 px。
- 保留现有浅薄荷色网格、深青文字和珊瑚橙强调色，不增加装饰性动态效果。

### SVG 手型

- 第一张站位卡维持现状。
- 第二张控制卡的左右手必须各自具有五根可辨识手指，并保留左手旋律、右手节奏的空间关系。
- 第三张录音卡使用结构清楚的双手大拇指向上和双手大拇指向下图形；每只手包含一个拇指和四根收拢手指，不能再出现三指或六指形态。
- SVG 使用确定性矢量路径绘制，不依赖运行时生成。
- 为测试增加语义标记，使测试能够验证每只手的手指数和手势类型。

## 2. 录音确认卡

- 移除可见的原生音频时间轴和拖动控件；保留隐藏的音频元素，仅供内部预览资源管理和下载逻辑复用。
- 主体改成静态录音状态模块：简化波形、录音已保存提示、TAKE 编号、时长和格式。
- 不增加自动播放，避免浏览器策略和现场扬声器状态造成误导。
- 底部按钮统一增大点击区域和字号，保留：
  - 确认并生成二维码
  - 重新录制
  - 下载到当前电脑
  - 放弃这段
- 手势语义保持现状：双手拇指向上确认，双手拇指向下取消并返回自由模式。

## 3. 分享卡与二维码海报

### 弹窗布局

- 桌面端分享状态使用紧凑双栏：左侧信息和链接，右侧海报预览。
- 弹窗整体限制在可用视口内，底部操作区固定保留在卡片内，不依赖用户滚动才能看到“放弃这段”。
- 窄屏改为纵向布局，并限制海报预览高度。

### 海报视觉

- 以现有 Bauhaus 手势音乐海报的构图为参考，保留左上二维码安全区、双手、音符阶梯和节奏网格等核心元素。
- 使用 `imagegen` 生成一张新模板，颜色改为：
  - 雾白与浅薄荷背景
  - 深青色主结构
  - 珊瑚橙强调
  - 少量暖黄色
  - 轻微纸张颗粒感
- AI 模板中不生成任何文字或二维码；所有动态文字和二维码由 Canvas 精确绘制，避免错字并保持可扫描性。
- 模板进入项目资产目录，不能只保留在临时生成目录。

### 海报文案

海报主要信息为：

- `WAIC 双手乐队`
- `欢迎打卡`
- `你是本场第 027 位音乐玩家`
- `TAKE 027`
- `扫码试听与下载 · 24H`

手机分享页同步显示活动编号。项目内部名称可保留在页面元数据中，但不再作为海报主标题。

## 4. 活动真实计数

### 计数语义

- 仅当用户确认录音且云服务器成功保存录音后分配编号。
- 重录、取消、上传失败不占用编号。
- 编号是当前活动计数，不与临时本地 TAKE 号混淆。

### 服务端实现

- 在现有云服务器录音服务中维护持久化活动计数器。
- 录音写入与编号分配在服务器端顺序执行；计数器采用临时文件写入后原子替换，避免半写状态。
- 服务器上传响应增加 `checkinNumber`，Netlify 代理原样返回，前端用于卡片、海报与分享页。
- 录音元数据写入活动编号，因此分享页可通过录音查询获得同一编号。
- 并发请求通过进程内串行队列保护。当前部署为单实例服务；若未来扩为多实例，需要迁移到支持原子递增的共享数据库。

### 重置

- 提供服务器端管理脚本或受签名保护的管理端点，可将计数器归零或设置指定起始值。
- 管理能力不出现在公共页面。
- 重置操作需要独立管理密钥，并记录重置后的值和时间。

## 5. Synthpop 场景

### Afterglow Coast

- BPM：118。
- 方向：明亮大调 Synthpop。
- 使用 16 步可循环主旋律，形成清晰的重复动机与向上推进感。
- 低音保持稳定四拍支撑，不与旋律争抢节奏。

### Blue Hour Drift

- BPM：120。
- 方向：带夜色感的小调 Synthpop，但不做慢速氛围音乐。
- 使用抓耳的短动机、适量切分和收束音，确保循环边界自然。
- 避免过大的随机跳进和连续空拍。

两段场景保持现有 ID 和展示名称，降低对场景切换、HUD 和既有测试的影响。

## 6. 测试与验收

### 自动测试

- SVG：验证第二、第三张卡每只手的手指数和手势标记。
- 录音 UI：验证可见原生音频控件已移除，状态模块和底部操作存在。
- 分享布局：验证共享状态的视口限制、海报尺寸和放弃按钮可见规则。
- 海报：验证新模板路径、二维码安全区、活动文案和编号格式。
- 计数器：验证顺序递增、并发不重复、上传失败不占号、重置、持久化和元数据读取。
- 音乐：验证两段场景 BPM、16 步序列以及有效音高范围。

### 浏览器验证

- 在本地浏览器检查三张指引卡、录音确认、二维码分享和窄屏布局。
- 使用测试录音完成一次确认、生成二维码、手机分享页读取编号的 smoke 流程。
- 检查控制台没有新增错误。

- 七项用户反馈全部有对应变更和测试。
- 全量测试通过。
- 测试分支生成新的 Netlify Draft 并完成在线 smoke。
- 生产主站和 `origin/main` 保持不变。
